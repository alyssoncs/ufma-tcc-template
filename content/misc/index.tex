\section{Código}

Frequentemente você vai precisar mencionar pequenos trechos de código no meio de um parágrafo, como o nome de um comando, de um arquivo ou de uma variável. Para isso, o comando \texttt{\textbackslash{}texttt} (de \foreign{typewriter text}) formata o seu argumento com uma fonte monoespaçada, sinalizando ao leitor que aquilo é código. Por exemplo, \texttt{\textbackslash{}texttt\{int x = 0;\}} produz \texttt{int x = 0;}. Repare, porém, que o \texttt{\textbackslash{}texttt} apenas troca a fonte: ele não conhece a linguagem do trecho e, portanto, não aplica realce de sintaxe.

Quando o trecho inline for código de verdade e o realce de sintaxe ajudar na leitura, prefira o comando \texttt{\textbackslash{}mintinline}, fornecido pelo pacote \href{https://ctan.org/pkg/minted}{minted}. Ele recebe a linguagem como primeiro argumento e o código delimitado por um par de caracteres iguais à sua escolha (por exemplo, barras verticais). Assim, escrever \mintinline{latex}@\mintinline{java}|int x = 0;|@ produz \mintinline{java}|int x = 0;|. Use o \texttt{\textbackslash{}texttt} para menções curtas e simples (nomes de comandos, arquivos e identificadores) e reserve o \texttt{\textbackslash{}mintinline} para trechos de código reais em que o realce agregar clareza.

Para trechos maiores, que merecem destaque próprio e uma legenda, use o ambiente \texttt{minted} dentro de um float \texttt{listing}. Como lembra \textcite{knuth:goto:1974}, programas devem ser escritos para humanos lerem. Utilize conforme o \autoref{lst:factorial-inline}.

\begin{listing}[htb]
  \inputminted{latex}{\CurrentFilePath/res/code/minted.tex}
  \caption{Inclusão de código}
  \label{lst:factorial-inline}
\end{listing}

O conteúdo dentro do ambiente \texttt{minted} será colocado dentro do seu documento, como os exemplos de código que você viu até agora, no caso do \autoref{lst:factorial-inline}, este iria gerar como output o \autoref{lst:factorial-inline-output}.

\begin{listing}[htb]
  \begin{minted}{java}
    public class Main {
        public static void main(String[] args) {
            System.out.println(factorial(10));
        }

        static int factorial(int n) {
            if (n == 1) return 1;

            return n * factorial(n - 1);
        }
    }
  \end{minted}
  \caption{Fatorial}
  \label{lst:factorial-inline-output}
\end{listing}

Caso queira deixar seu código \LaTeX{} mais compacto, você pode colocar o seu código em um arquivo separado e incluir em seu documento com o comando \texttt{\textbackslash{}inputminted}. Esse \template{} sugere que coloque esse códigos em \contentdir{}\texttt{\{nome-da-seção\}/}\codedir. Assumindo que o código existe em um arquivo \texttt{Factorial.java} dentro desse diretório, o exemplo anterior poderia ser escrito conforme o \autoref{lst:factorial-include}.

\begin{listing}[htb]
  \inputminted{latex}{\CurrentFilePath/res/code/factorial-include.tex}
  \caption{Inclusão de código através de um arquivo externo}
  \label{lst:factorial-include}
\end{listing}

\subsection{Linguagens e estilos}

A linguagem do código é informada como argumento do ambiente \texttt{minted} (o \emph{lexer} do Pygments, como \texttt{java}, \texttt{latex} ou \texttt{python}), o que dá o realce correto de palavras reservadas, strings e comentários sem precisar cadastrá-las à mão. O tema de cores é global e definido no arquivo \pkgfile, com o comando \texttt{\textbackslash{}usemintedstyle}; opções padrão (moldura, numeração, fonte) ficam no \texttt{\textbackslash{}setminted}. Consulte a documentação oficial para coisas mais complexas.

\section{Remissões internas} \label{sec:cross-ref}

Se você quer referenciar partes do seu documento, como: capítulos, seções, subseções, figuras, tabelas e códigos, você deve primeiramente adicionar uma \labelterm{} nesse elemento, depois deve usar o comando \texttt{\textbackslash{}autoref} ou \texttt{\textbackslash{}ref} no local que deseja mencioná-lo.

O comando \texttt{\textbackslash{}autoref} adiciona um link para o elemento contendo o nome que o identifica (como: \enquote{capítulo}, \enquote{seção}, \enquote{subseção}, \enquote{tabela} e etc.) seguido do número desse elemento no documento.

O comando \texttt{\textbackslash{}ref} adiciona um link para o elemento contendo \emph{apenas} o número do elemento no documento.

Como exemplo, adicionei a \labelterm{} \enquote{\texttt{sec:cross-ref}} na seção atual, ao utilizar o comando \texttt{\textbackslash{}autoref\{sec:cross-ref\}}, vou obter o seguinte resultado: \enquote{\autoref{sec:cross-ref}}.
Entretanto, se utilizar \texttt{\textbackslash{}ref\{sec:cross-ref\}}, vou obter apenas: \enquote{\ref{sec:cross-ref}}, que pode ser útil ao referenciar múltiplos elementos ao mesmo tempo.

A forma de adicionar \labelterm[s] é ligeiramente diferente dependendo do tipo de elemento sendo referenciado.

\subsection{\texorpdfstring{\labelterm*[s]}{Labels} em capítulos, seções e subseções}

Para capítulos, seções e subseções, usamos o comando \texttt{label} seguido do id da \labelterm{} que queremos atribuir, conforme o \autoref{lst:chap-sec-subsec-label}.

\begin{listing}[htb]
  \begin{minted}{latex}
    \chapter{Introdução} \label{chap:intro}

    Blá.

    \section{Panorama Geral} \label{sec:overview}

    Blá.

    \subsection{Questões Fundamentais} \label{sub:fundamental-questions}

    Blá.
  \end{minted}
  \caption{Atribuição de \labelterm[s] em capítulos, seções e subseções}
  \label{lst:chap-sec-subsec-label}
\end{listing}

Desta forma, eu posso referenciar o capítulo \enquote{Introdução} usando o comando: \texttt{\textbackslash{}autoref\{chap:intro\}}.

\subsection{\texorpdfstring{\labelterm*[s]}{Labels} em figuras}

Também se utiliza o comando \texttt{label}, entretanto, dentro do ambiente \texttt{figure} e após a macro \texttt{caption}, consulte o \autoref{lst:figure}.

\subsection{\texorpdfstring{\labelterm*[s]}{Labels} em tabelas}

Também se utiliza o comando \texttt{label}, entretanto, dentro do primeiro bloco do comando \texttt{IBGEtab} e após a macro \texttt{caption}, consulte o \autoref{lst:tab}.

\subsection{\texorpdfstring{\labelterm*[s]}{Labels} em códigos}

Assim como em figuras e tabelas, usamos o comando \texttt{label} dentro do float \texttt{listing} (criado pelo \texttt{minted}), logo após a macro \texttt{caption}, consulte os códigos~\ref{lst:factorial-inline} e~\ref{lst:factorial-include}.

\section{Aspas}

Aspas em \LaTeX\ são mais complicadas do que parecem. Ao tentar usar os caracteres \enquote{\texttt{\textquotesingle}} ou \enquote{\texttt{"}} (aspas simples e dupla, respectivamente), o resultado não será o esperado, não gerando as aspas curvas. % chktex 18
O uso errôneo desse caracteres no \autoref{lst:wrong-quotation} irá gerar o seguinte resultado:
'Aspas simples' e "aspas duplas". % chktex 18 % chktex 32

\begin{listing}[htb]
  \begin{minted}{latex}
    'Aspas simples' e "aspas duplas".
  \end{minted}
  \caption{Uso incorreto de aspas}
  \label{lst:wrong-quotation}
\end{listing}

Note que \enquote{\texttt{\textquotesingle}} sempre gera aspas direitas e \enquote{\texttt{"}} sempre gera aspas duplas retas. % chktex 18
Para produzir aspas corretamente, o \LaTeX\ espera que se utilize os símbolos presentes na \autoref{tab:quotes}.

\begin{table}[htb]
  \begin{center}
    \IBGEtab{
      \caption{Uso de aspas no \LaTeX}
      \label{tab:quotes}
    }{
      \begin{tabular}{lclc}
        \toprule
        Resultado desejado    & Símbolo                                   & Descrição do símbolo & Resultado \\
        \midrule
        Aspa simples esquerda & \texttt{\`{}}                             & 1 acento grave       & `         \\
        Aspa simples direita  & \texttt{\textquotesingle}                 & 1 aspa simples       & '         \\
        Aspa dupla esquerda   & \texttt{\`{}\`{}}                         & 2 acentos grave      & ``        \\
        Aspa dupla direita    & \texttt{\textquotesingle\textquotesingle} & 2 aspas simples      & ''        \\
        \bottomrule
      \end{tabular}
    }{
      \fonte{\autoriapropria}
    }
  \end{center}
\end{table}

Ou seja, o exemplo anterior deveria ser escrito conforme o \autoref{lst:correct-quotation}, gerando:
`Aspas simples' e ``aspas duplas''.

\begin{listing}[htb]
  \begin{minted}{latex}
    `Aspas simples' e ``aspas duplas''.
  \end{minted}
  \caption{Uso correto de aspas}
  \label{lst:correct-quotation}
\end{listing}

Como esse método de inclusão de aspas é pouco intuitivo e fácil de errar, podemos usar o pacote \href{https://ctan.org/pkg/csquotes}{csquotes} para auxiliar. Usamos os comando \texttt{enquote*} para aspas simples e \texttt{enquote} para aspas duplas.
O exemplo pode ser então reescrito conforme o \autoref{lst:csquotes}, gerando:
\enquote*{Aspas simples} e \enquote{Aspas duplas}.

\begin{listing}[htb]
  \begin{minted}{latex}
    \enquote*{Aspas simples} e \enquote{Aspas duplas}.
  \end{minted}
  \caption{Uso do pacote csquotes}
  \label{lst:csquotes}
\end{listing}

Se o seu texto utiliza com bastante frequência termos que você quer que tenham tipografia e escrita consistente, elas podem ser adicionadas como macros no arquivo \macrofile, com o comando \texttt{newterm}. Vamos olhar alguns exemplos no \autoref{lst:newterm}.

\begin{listing}[htb]

    \newterm{\api}{API}
    \newterm{\broadcast}{\foreign{broadcast}}
    \newterm{\printf}{\texttt{printf}}
  \end{minted}

Neste caso, foram definidos 3 macros (\texttt{\textbackslash{}api}, \texttt{\textbackslash{}broadcast} e \texttt{\textbackslash{}printf}), a intenção do autor é que o termo \api{} sempre seja escrito em maiúsculo, o termo \broadcast{} sempre seja escrito como uma palavra estrangeira e o termo \printf{} sempre seja formatado com fonte monoespaçada.

Repare nas chaves vazias (\texttt{\textbackslash{}broadcast\{\}}) usadas quando o termo é seguido de um espaço. Como qualquer comando \LaTeX\ sem argumentos, um termo \enquote{engole} o espaço que vem logo depois dele: escrever \texttt{\textbackslash{}broadcast de} produziria \enquote{\broadcast de}, sem o espaço. As chaves vazias delimitam o fim do comando e preservam o espaço seguinte. Quando o termo é seguido de pontuação --- como em \texttt{\textbackslash{}api,} --- as chaves não são necessárias. O projeto também roda um linter (via \texttt{just lint}), que alerta sobre termos seguidos de espaço sem as chaves. % chktex 1

Como alternativa às chaves vazias, a sequência \texttt{\textbackslash{}broadcast\textbackslash{} de dados} (barra invertida seguida de espaço) também preserva o espaço após o termo, gerando: \broadcast\ de dados.

\subsection{Plural e capitalização}

Um mesmo termo costuma aparecer no singular, no plural e com a primeira letra maiúscula (no início de uma frase, por exemplo). Em vez de criar uma macro para cada variação, o \texttt{newterm} aceita chaves opcionais e dois modificadores de uso, conforme o \autoref{lst:newterm-plural}.

\begin{listing}[htb]
  \begin{minted}{latex}
    \newterm{\vetor}{vetor}[plural=vetores]
    \newterm{\framework}{\foreign{framework}}[plural=\foreign{frameworks}]
    \newterm{\justica}{justiça}
    \newterm{\id}{id}[plural=ids, cap=ID, capplural=IDs]
  \end{minted}
  \caption{Definição de termos com plural e capitalização}
  \label{lst:newterm-plural}
\end{listing}

A opção \texttt{plural} define a forma do termo no plural. Sem ela, o termo não tem plural e o modificador \texttt{[s]} interrompe a compilação --- útil para termos incontáveis, como \texttt{\textbackslash{}justica} no \autoref{lst:newterm-plural}.

As opções \texttt{cap} e \texttt{capplural} permitem fornecer formas capitalizadas explícitas, necessárias quando a capitalização automática não produz o resultado desejado. Quando omitidas, a primeira letra do singular e do plural é capitalizada automaticamente, preservando qualquer formatação (a capitalização automática atravessa comandos como \texttt{\textbackslash{}foreign} ou \texttt{\textbackslash{}texttt} e capitaliza apenas a primeira letra do texto).
O termo \framework, definido no \macrofile, ilustra o caso automático preservando o itálico:
\begin{alineas}
  \item \texttt{\textbackslash{}framework} produz \enquote{\framework};
  \item \texttt{\textbackslash{}framework*} produz \enquote{\framework*};
  \item \texttt{\textbackslash{}framework[s]} produz \enquote{\framework[s]};
  \item \texttt{\textbackslash{}framework*[s]} produz \enquote{\framework*[s]}.
\end{alineas}

Já o termo \texttt{\textbackslash{}id} do \autoref{lst:newterm-plural} precisa das opções, a forma automática seria \enquote{Id} e \enquote{Ids}, mas \texttt{cap} e \texttt{capplural} fixam as siglas corretas \enquote{\id*} e \enquote{\id*[s]}.

No uso, o asterisco (\texttt{*}) produz a versão capitalizada e o argumento opcional \texttt{[s]} produz a versão no plural; os dois podem ser combinados. O comando ainda age como um validador estrito: pedir o plural de um termo que não o define interrompe a compilação. O \autoref{lst:newterm-plural-usage} reúne esses casos.

\begin{listing}[htb]
  \begin{minted}{latex}
    \vetor       % vetor
    \vetor[s]    % vetores
    \vetor*      % Vetor
    \vetor*[s]   % Vetores
    \framework   % framework (em itálico)
    \framework*  % Framework (capitaliza preservando o itálico)
    \justica     % justiça
    \justica*    % Justiça
    \justica[s]  % quebra a build: \justica não define plural
    \id          % id
    \id[s]       % ids
    \id*         % ID  (sem cap, sairia "Id")
    \id*[s]      % IDs (sem capplural, sairia "Ids")
  \end{minted}
  \caption{Uso dos modificadores de plural e capitalização}
  \label{lst:newterm-plural-usage}
\end{listing}

O asterisco também pode ser usado na \emph{definição} do termo, com um papel diferente: \texttt{\textbackslash{}newterm*} cria um termo cuja capitalização fica \emph{bloqueada}, útil para nomes que nunca devem começar com maiúscula. O \autoref{lst:newterm-blocked} define dois termos assim: \texttt{\textbackslash{}iphone}, cujo \enquote{i} minúsculo faz parte da marca, e \texttt{\textbackslash{}pdfextension}, uma extensão de arquivo.

\begin{listing}[htb]
  \begin{minted}{latex}
    \newterm*{\iphone}{iPhone}[plural=iPhones]
    \newterm*{\pdfextension}{\texttt{.pdf}}
  \end{minted}
  \caption{Definição de termos com capitalização bloqueada}
  \label{lst:newterm-blocked}
\end{listing}

Com isso, \texttt{\textbackslash{}iphone} e \texttt{\textbackslash{}iphone[s]} produzem \enquote{\iphone} e \enquote{\iphone[s]} normalmente, mas \texttt{\textbackslash{}iphone*} interrompe a compilação com um erro, impedindo a forma capitalizada. Da mesma forma, passar as opções \texttt{cap} ou \texttt{capplural} a um termo criado com \texttt{\textbackslash{}newterm*} é contraditório e falha já na definição. Esse rigor evita que variações inexistentes passem despercebidas no texto final.

\subsection{Termos aninhados}

O texto de um termo pode conter outros termos, inclusive com seus próprios modificadores de plural e capitalização. No \autoref{lst:newterm-nested}, o termo \texttt{\textbackslash{}tabelaponteiros} reaproveita o termo \texttt{\textbackslash{}ponteiro} na definição do singular e do plural.

\begin{listing}[htb]
  \begin{minted}{latex}
    \newterm{\ponteiro}{ponteiro}[plural=ponteiros]
    \newterm{\tabelaponteiros}{tabela de \ponteiro[s]}[plural=tabelas de \ponteiro[s]]
  \end{minted}
  \caption{Definição de termos aninhados}
  \label{lst:newterm-nested}
\end{listing}

A capitalização aninhada também funciona: \texttt{\textbackslash{}tabelaponteiros} resulta em \enquote{\tabelaponteiros}, \texttt{\textbackslash{}tabelaponteiros[s]} em \enquote{\tabelaponteiros[s]} e \texttt{\textbackslash{}tabelaponteiros*} em \enquote{\tabelaponteiros*}, capitalizando apenas a primeira letra e preservando o termo interno.

\section{Unidades de medida}

Aconselho o uso do pacote \href{https://ctan.org/pkg/siunitx}{siunitx}, a leitura do manual é extremamente útil para descobrir todos os comandos que você vai precisar usar. Aqui está um resumo:

\subsection{Unidades}

Use \texttt{unit} para escrever unidades de medida. Esse comando aceita duas sintaxes, uma usando macros predefinidas, e outra usando strings. Por exemplo, para escrever \enquote{metros por segundo ao quadrado}, podemos usar \verb|\unit{\meter\per\second\squared}| ou \verb|\unit{m/s^2}|. Entretanto, esses dois comandos vão produzir resultados levemente diferente, conforme

\begin{table}[htb]
  \begin{center}
    \IBGEtab{
      \caption{Especificando unidades de medida}
      \label{tab:unit}
    }{
      \begin{tabular}{lc}
        \toprule
        Comando                                                                                                                & Resultado                        \\
        \midrule
        \texttt{\textbackslash{}unit\{\textbackslash{}meter\textbackslash{}per\textbackslash{}second\textbackslash{}squared\}} & \unit{\meter\per\second\squared} \\
        \texttt{\textbackslash{}unit\{m/s\textasciicircum{}2\}}                                                                & \unit{m/s^2}                     \\
        \bottomrule
      \end{tabular}
    }{
      \fonte{\autoriapropria}
    }
  \end{center}
\end{table}

Apesar de representar a mesma unidade, por padrão, o macro  \verb|\per| usa o formato de exponencial, isso pode ser alterado passando um argumento opcional, da seguinte forma: \verb|\unit[per-mode=symbol]{\meter\per\second\squared}|, gerando: \unit[per-mode=symbol]{\meter\per\second\squared}.

Também existem alguns macros abreviados, no caso anterior, poderíamos escrever
\verb|\unit[per-mode=symbol]{\m\per\s\squared}|, para obter: \unit[per-mode=symbol]{\m\per\s\squared}.

Os prefixos como quilo, mega, e giga também estão definidos e podem ser usados normalmente, para obter
\unit{\kilo\gram}, usa-se \verb|\unit{\kilo\gram}|.

Prefixos binários também estão disponíveis (kibi, mebi, gibi, etc), assim como macros para bit e byte, por exemplo: \verb|\unit[per-mode=symbol]{\mebi\byte\per\second}|, produz \unit[per-mode=symbol]{\mebi\byte\per\second}.

\subsection{Quantidades}

Quantidades podem ser especificadas com a macro \texttt{qty}, essa macro aceita dois parâmetros obrigatórios, o número e a unidade. Para especificar \enquote{5 gibibytes}, você usaria \verb|\qty{5}{\gibi\byte}|, que produz: \qty{5}{\gibi\byte}.

\section{Fonte}

A \abnt{} não determina um tipo de fonte específico, mas é um equívoco comum que deve ser utilizado a fonte Arial. Se você precisar mudar a fonte, esse documento é compilado com \XeLaTeX\ e usa o pacote \href{https://ctan.org/pkg/fontspec}{fontspec}, que permite usar qualquer fonte instalada no seu computador através do comando \mintinline{latex}|\setmainfont{Nome da Fonte}|.
